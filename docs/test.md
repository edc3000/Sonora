# Sonora 产品测试标准

本文档定义 Sonora 网页端和桌面端的产品级测试标准。目标不是追求理论上的零缺陷，而是确保用户能稳定、顺畅、可信地使用一个私人 AI 音乐电台：打开即可听，长期播放不断，AI 主持有氛围，外部服务失败时也能继续使用。

## 产品定位

Sonora 的重点输出形态是：

- 网页端：低门槛访问、快速配置、即时进入私人 AI 电台。
- 桌面端：长期常驻、后台播放、沉浸式听歌和系统级稳定体验。

核心验收口径：

> 网页端让用户快速进入私人 AI 电台；桌面端让电台像一个常驻音乐伴侣一样稳定运行。无论模型、TTS、网易云音乐服务是否偶发失败，用户都不应该失去“继续听歌”的能力。

## 发布准入标准

| 维度 | 测试通过目标 |
| :--- | :--- |
| 阻塞问题 | P0/P1 bug 数量为 0 |
| 核心链路 | 打开产品 -> 配置服务 -> 登录网易云 -> 同步歌单 -> 生成电台 -> 播放歌曲 -> AI 主持播报，全流程通过 |
| 首次可听 | 正常配置下 1 分钟内进入可播放状态；无外部服务时也能用兜底歌曲进入可听状态 |
| 操作响应 | 播放、暂停、切歌、打开设置、发送自然语言指令，UI 响应小于 200ms |
| 播放稳定性 | 网页端连续播放 30 分钟不中断；桌面端连续播放 2 小时不中断 |
| 队列稳定性 | 播放过程中队列不空转、不重复异常、不出现当前歌曲和实际音频不一致 |
| AI 体验 | 推荐、主持词、过渡语不空、不乱码、不模板化，能体现用户歌单、上下文和听歌记忆 |
| 降级能力 | NCM、LLM、TTS 任一失败时，产品仍能播放、可操作、可恢复 |
| 隐私安全 | API Key、登录态、`.env`、用户歌单缓存不在前端明文泄露，不被提交到 Git |
| 视觉质量 | 无白屏、无明显布局错位、无文本溢出、无按钮不可点 |

## 核心用户旅程测试

| 场景 | 测试内容 | 测试通过目标 |
| :--- | :--- | :--- |
| 新用户首次体验 | 打开产品，配置模型、TTS、NCM 服务，登录网易云并同步歌单 | 用户不需要读代码或手动改文件即可完成主流程配置 |
| 首次生成电台 | 同步完成后启动电台 | 能生成当前歌曲和后续队列，并开始播放第一首歌 |
| 日常使用 | 二次打开产品 | 能恢复最近合理的播放状态、队列、偏好和设置 |
| 自然语言控制 | 输入“来点粤语”“换一首”“暂停”“继续播放”等 | 控制类指令直接执行，音乐类指令进入推荐和选歌链路 |
| AI 主持体验 | 每首歌前生成 intro，首歌前生成 say，切换时生成 segue | 文案语言匹配歌曲，不空、不乱码、有具体歌曲锚点和氛围 |
| 异常恢复 | 模型失败、TTS 失败、网易云链接失效、网络中断 | 用户看到明确状态，产品不白屏、不卡死，恢复后可继续听 |

## 网页端测试项

| 测试项 | 测试内容 | 测试通过目标 |
| :--- | :--- | :--- |
| 首屏加载 | 打开首页 | 3 秒内可见播放器主体；无白屏、无致命控制台错误 |
| 浏览器兼容 | Chrome、Safari、Edge 主流程测试 | 播放、设置、登录、同步、点歌、切歌均可用 |
| PWA 安装 | 安装为 PWA 并从桌面图标启动 | 图标、名称、启动页、主题色正常；启动后主流程可用 |
| 移动端布局 | iPhone/Android 常见尺寸测试 | 播放器、队列、设置抽屉、二维码不遮挡、不溢出 |
| 音频播放 | 播放、暂停、拖动进度、切换歌曲 | UI 状态和真实音频一致；切歌不残留上一首音频 |
| 自动播放限制 | 浏览器阻止自动播放时打开页面 | 有明确播放入口，不陷入静默失败或无限 loading |
| 刷新恢复 | 播放中刷新页面 | 当前歌曲、队列、播放状态恢复合理，不损坏状态 |
| WebSocket 同步 | 播放、切歌、队列变化、主持状态推送 | 前端实时更新，无重复事件造成 UI 抖动 |
| 多标签页 | 同时打开两个页面 | 至少不会造成队列损坏、播放状态错乱或接口风暴 |
| 设置保存 | 保存 OpenAI、TTS、NCM 配置 | 保存后立即生效；密钥只显示配置状态或尾号 |
| 弱网场景 | 接口超时、断网、恢复网络 | 页面不崩溃，有错误提示或可恢复状态 |

## 桌面端测试项

桌面端如果使用 Electron、Tauri 或 PWA 包装，需要额外验证常驻、后台和系统交互能力。

| 测试项 | 测试内容 | 测试通过目标 |
| :--- | :--- | :--- |
| 安装启动 | 首次安装、首次启动、二次启动 | 应用能稳定启动，不依赖开发环境命令 |
| 窗口行为 | 缩放、最小化、恢复、全屏 | UI 不错位，播放不中断，状态不丢失 |
| 后台播放 | 窗口最小化或切到后台 | 音乐继续播放，回到前台后状态一致 |
| 关闭语义 | 点击关闭、退出应用、托盘常驻 | 关闭窗口和退出应用的行为清晰、可预期 |
| 系统音频 | 调整系统音量、切换蓝牙耳机、切换输出设备 | 播放不中断或能自动恢复，不出现无声但 UI 显示播放 |
| 本地数据路径 | 配置、缓存、用户数据读写 | 路径稳定，不污染系统目录，不误删用户数据 |
| 重启恢复 | 应用崩溃、系统重启、服务重启 | 能恢复到最近合理状态，不要求用户重新配置 |
| 版本更新 | 新版本覆盖旧版本 | 旧配置、旧登录态、旧缓存、旧用户数据不损坏 |
| 长时间播放 | 连续播放 2 小时 | 无明显内存持续上涨，队列不断，切歌正常 |
| 离线/弱网 | 离线启动、弱网播放、网络恢复 | 有可解释状态，网络恢复后可继续播放或重新拉取音频 |

## AI 电台能力测试项

| 测试项 | 测试内容 | 测试通过目标 |
| :--- | :--- | :--- |
| 用户画像 | 同步网易云喜欢列表、歌单、歌手和语言分布 | 能生成可用的听歌画像，不因空字段或重复数据失败 |
| 推荐选歌 | 根据用户输入、偏好、时间、天气、最近播放生成候选 | 至少返回可播放歌曲；避免频繁重复最近播放 |
| 多样性 | 连续生成多个队列 | 不连续重复同一歌手；不总是落在同一小撮高频歌手上 |
| 语言匹配 | 中文、粤语、英文歌曲混合播放 | 主持词语言与歌曲语言基本匹配，专名保留原文 |
| 主持词质量 | 检查 say、intro、segue | 有具体歌曲锚点、听感细节或场景感；避免空泛模板话 |
| TTS 生成 | 正常 TTS 和 TTS 失败场景 | 正常时播放缓存音频；失败时浏览器语音兜底或静默降级明确 |
| 兜底推荐 | 未配置模型或模型调用失败 | 仍能用本地规则生成可播放队列 |

## 音频与队列测试项

| 测试项 | 测试内容 | 测试通过目标 |
| :--- | :--- | :--- |
| 播放链接 | 获取网易云播放链接 | 可播放；链接失效时能刷新或切换下一首 |
| Range 请求 | 拖动进度条、从中间播放 | 音频服务支持 Range，不整首卡死重载 |
| 切歌一致性 | 快速连续点击下一首/上一首 | UI 当前歌曲、实际音频、队列状态一致 |
| 队列补全 | 播放到队列末尾 | 自动补队列或进入明确结束状态，不空转 |
| 历史记录 | 播放多首后查看历史 | 最近播放记录准确，用于后续去重和推荐 |
| 缓存控制 | TTS mp3 缓存命中和失效 | 命中时不重复请求；失效时可重新生成 |

## 安全与隐私测试项

| 测试项 | 测试内容 | 测试通过目标 |
| :--- | :--- | :--- |
| 前端密钥暴露 | 检查设置接口和页面源码 | 不返回完整 API Key、cookie、登录态 |
| 日志泄露 | 检查服务端日志、浏览器控制台 | 不打印完整密钥、二维码敏感结果、用户 cookie |
| Git 提交检查 | 检查工作区和暂存区 | `.env`、登录态、缓存音频、用户私有数据不被提交 |
| 本地数据权限 | 检查用户数据、缓存、状态文件 | 只保存运行所需数据，路径明确，可清理 |
| 配置写入 | 从 UI 保存配置 | 写入内容正确，不覆盖无关配置，不生成明文泄露页面 |

## Bug 分级标准

| 等级 | 定义 | 发布要求 |
| :--- | :--- | :--- |
| P0 | 无法启动、无法播放、密钥泄露、用户数据损坏、桌面端无法打开 | 必须修复，不允许发布 |
| P1 | 登录、同步、播放、切歌、AI 生成任一核心链路中断 | 必须修复，不允许发布 |
| P2 | 移动端明显错位、TTS 偶发失败、文案质量明显差、弱网恢复差 | 发布前优先修复 |
| P3 | 小文案、小样式、小概率边缘问题 | 可记录到后续迭代 |

## 回归测试建议

以下改动必须触发完整核心链路回归：

- 修改播放器、队列、音频代理、WebSocket 状态同步。
- 修改 NCM 登录、歌单同步、播放链接获取。
- 修改 Agent 选歌、上下文构建、主持词 prompt。
- 修改 TTS 管线、缓存策略、浏览器语音兜底。
- 修改设置页、配置保存、密钥读取逻辑。
- 打包桌面端、更新运行时、变更本地数据路径。

每次发布前至少完成：

- 网页端核心用户旅程 1 轮。
- 桌面端连续播放 2 小时。
- NCM、LLM、TTS 分别失败的降级测试。
- Git 暂存区和前端接口的密钥泄露检查。
